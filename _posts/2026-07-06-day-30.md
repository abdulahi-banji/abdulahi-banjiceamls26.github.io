  I came into the lab pretty early today studying Python DSA, solving problems and studying for an hour. After my teammates came in, we analyzed the results of the YOLO11x training from yesterday. Apparently the training stopped with a message: "EarlyStopping: Training stopped early as no improvement was observed in the last 100 epochs." This was a new message to me, and it was interesting to see how the model is able to identify whether or not it's learning.

  After that, we had a long session with our faculty mentor where he explained the code for the robot arm, camera QR code detection, and Lego piece detection and pick. As he walked through the code, I learned a lot about the functions, and he also explained things like the transformation matrix and why it's important. We tried debugging the code, and I highlighted the useful debugging skill of using multiple print statements to see where the problem is.

  After we found the problem, we decided to make the QR code less complex. We tried using online QR code generators, but we weren't getting the desired results. Then I suggested we just run the Python code for generating QR codes, since it was pretty easy. After that, we made significant progress in collaboration between the camera and the robot arm, and the camera is now able to detect the QR code and identify the Lego pieces. A problem we still need to work on is reducing the size of the QR code, as well as taking intentional and careful note of the orientation of the piece.

  After that, we decided to train our model using YOLOv8 P2, this time with the medium variant. I then worked a bit on my literature review section on Overleaf.
